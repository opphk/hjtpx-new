# CaptchaX Electron SDK

CaptchaX Electron SDK 是一个功能强大的跨平台桌面验证码解决方案，专为 Electron 桌面应用设计。该 SDK 提供了多种验证码类型，包括滑块验证码、点选验证码、拼图验证码、旋转验证码、文字验证码和图标验证码，能够有效防止自动化攻击和机器人行为。

## 项目概述

CaptchaX Electron SDK 旨在为桌面应用程序提供企业级的验证码验证功能。通过集成该 SDK，开发者可以轻松地在 Electron 应用中实现用户身份验证，保护应用程序的核心功能免受恶意自动化工具的侵害。该 SDK 支持 Windows、macOS 和 Linux 三大主流操作系统，充分利用 Electron 的跨平台特性，为用户提供一致的体验。

该 SDK 的核心优势在于其模块化设计和灵活的 API 接口。开发者可以根据实际需求选择合适的验证码类型，并通过简洁的 API 调用实现验证码的获取、验证和管理。同时，SDK 内置了完善的事件系统和日志记录功能，便于开发者进行调试和问题排查。

## 核心特性

CaptchaX Electron SDK 提供了丰富的功能和特性，使其成为桌面应用验证码解决方案的理想选择。

### 多类型验证码支持

该 SDK 支持六种不同类型的验证码，每种类型都有其独特的验证机制和应用场景。滑块验证码通过检测用户的滑动轨迹来判断是否为真实用户，这种验证方式用户体验友好且安全性较高。点选验证码要求用户点击图像中指定的区域，适合需要用户识别特定内容的场景。拼图验证码需要用户将拼图块滑动到正确位置，结合了交互性和安全性。旋转验证码要求用户将图像旋转到指定角度，操作直观且难以被自动化程序模拟。文字验证码显示字符或数字序列，用户需要正确输入，经典且可靠。图标验证码显示多个图标，用户需要根据提示选择符合条件的所有图标，交互性强且趣味性高。

### 跨平台兼容性

CaptchaX Electron SDK 基于 Electron 框架开发，能够在 Windows、macOS 和 Linux 系统上运行。SDK 自动适配不同平台的系统特性，包括窗口管理、通知系统和菜单栏等。在 Windows 平台上，SDK 支持系统托盘功能和任务栏通知。在 macOS 平台上，SDK 完美集成了 Dock 菜单和通知中心。在 Linux 平台上，SDK 支持多种桌面环境的系统通知。

### 完善的 API 设计

SDK 提供了简洁而强大的 API 接口，开发者可以通过链式调用轻松配置和管理验证码。所有的 API 调用都支持异步操作，不会阻塞主线程。SDK 内置了自动重试机制，当网络请求失败时能够自动进行重试，提高了应用的稳定性。同时，SDK 提供了丰富的事件回调，开发者可以精确控制验证码的各个阶段。

### 原生系统集成

除了核心的验证码功能外，CaptchaX Electron SDK 还深度集成了 Electron 的原生特性。SDK 支持系统托盘运行，应用可以在后台持续运行而不占用任务栏空间。SDK 支持原生系统通知，验证成功或失败时可以向用户发送系统通知。SDK 支持原生菜单栏，可以通过菜单快速访问各种功能。此外，SDK 还支持快捷键绑定，方便用户进行快捷操作。

## 安装指南

在开始使用 CaptchaX Electron SDK 之前，需要确保开发环境满足以下要求。首先，需要安装 Node.js 环境，建议使用 Node.js 18.0 或更高版本以获得最佳性能和安全性。其次，需要安装 Electron 框架，SDK 已经将 Electron 作为开发依赖包含在 package.json 中。最后，需要一个包管理器来安装项目依赖，推荐使用 npm 或 yarn。

安装步骤如下：首先，克隆或下载 CaptchaX Electron SDK 的源代码到本地目录。然后，进入项目根目录并运行包管理器安装依赖。在安装完成后，即可开始使用 SDK 进行开发。如果需要打包应用程序，可以运行构建命令生成各平台的安装包。

项目目录结构设计清晰，便于理解和维护。核心 SDK 代码位于项目根目录的 captcha-electron.js 文件中，该文件包含了主要的 SDK 类和函数。组件代码位于 src/components 目录下，每个验证码类型都有独立的组件文件。主进程代码位于 src/main 目录下，负责处理 Electron 的核心功能和系统集成。渲染进程代码位于 src/renderer 目录下，包含用户界面和交互逻辑。示例应用位于 example 目录下，提供了完整的使用示例参考。

## 快速开始

以下是一个简单的示例，展示了如何使用 CaptchaX Electron SDK 获取和验证滑块验证码。

首先，需要在项目中引入 SDK 并创建实例。创建实例时可以传入配置选项，如 API 地址、超时时间等。SDK 支持链式调用，可以通过点号连接多个方法调用。接下来，可以调用 getCaptcha 方法获取验证码，该方法接收验证码类型参数并返回验证码数据。然后，可以使用返回的数据初始化验证码组件进行展示。最后，当用户完成验证操作后，调用 verify 方法提交验证请求。

SDK 的使用流程设计得非常直观，开发者只需要关注业务逻辑而不需要关心底层实现细节。每个步骤都有清晰的回调和数据返回，便于进行状态管理和界面更新。

## API 参考

### SDK 初始化

CaptchaElectronSDK 是 SDK 的核心类，提供了所有的验证功能。通过 createSDK 工厂函数可以创建 SDK 实例，创建时可以传入配置对象来自定义 SDK 的行为。配置选项包括 API 地址、请求超时时间、重试次数、是否启用通知等功能开关。

SDK 实例提供了丰富的方法来获取和验证验证码。getCaptcha 方法用于获取指定类型的验证码，返回验证码数据对象。verify 方法用于提交验证结果，接收验证码类型、ID、轨迹数据和用户响应参数。isExpired 方法用于检查当前验证码是否已过期。此外，SDK 还为每种验证码类型提供了便捷的快捷方法，如 slider、click、puzzle 等方法可以直接获取对应类型的验证码。

### 事件系统

SDK 内置了完善的事件系统，开发者可以监听各种事件来实现自定义逻辑。READY 事件在验证码加载完成时触发，SUCCESS 事件在验证成功时触发，FAIL 事件在验证失败时触发，ERROR 事件在发生错误时触发，EXPIRED 事件在验证码过期时触发，CLOSE 事件在验证码关闭时触发。

通过 on 方法可以注册事件监听器，通过 off 方法可以移除事件监听器。事件回调函数会接收到相关的数据对象，如验证结果、错误信息等。事件系统采用了 Node.js 的 EventEmitter 模式，开发者可以轻松实现复杂的业务逻辑。

### 配置选项

SDK 提供了多种配置选项来自定义其行为。apiBaseUrl 选项用于设置 API 服务地址，默认指向本地开发服务器。timeout 选项用于设置请求超时时间，单位为毫秒，默认值为 10000 毫秒。retryAttempts 选项用于设置请求失败时的最大重试次数，默认值为 3 次。enableNotifications 选项用于控制是否启用系统通知功能。enableTray 选项用于控制是否启用系统托盘功能。

除了 SDK 全局配置外，每个验证码组件也提供了丰富的配置选项。width 和 height 选项用于设置组件的尺寸。backgroundImage 和 sliderImage 选项用于设置验证码的背景图片。clickLimit 和 selectLimit 选项用于设置点击和选择的数量限制。各种配置选项都提供了合理的默认值，开发者可以根据需要进行自定义。

## 验证码组件

### 滑块验证码组件

SliderCaptcha 组件实现了滑块拖动验证功能。用户需要将滑块拖动到正确的位置来完成验证。组件提供了完整的用户界面，包括背景图片显示区、滑块轨道和滑块按钮。组件会自动记录用户的滑动轨迹，包括滑动位置和时间戳，这些数据可以用于后端的风控分析。

组件支持鼠标和触摸两种交互方式，能够在不同类型的设备上正常工作。组件提供了丰富的事件回调，包括 verify、ready、refresh 等事件。组件还提供了 showSuccess 和 showError 方法来显示验证结果，提供了 reset 方法来重置验证码状态。

### 点选验证码组件

ClickCaptcha 组件实现了图像点选验证功能。用户需要点击图像中指定的目标区域来完成验证。组件会在用户点击时显示序号标记，帮助用户记住已点击的位置。组件支持自定义点击数量限制，默认需要点击 4 个目标。

组件会自动跟踪点击位置和时间戳，并计算点击的准确性。组件提供了可视化指示器显示当前已点击数量。组件支持验证触发，当用户完成所有点击后可以自动或手动提交验证请求。

### 拼图验证码组件

PuzzleCaptcha 组件实现了拼图滑动验证功能。与滑块验证码不同，拼图验证码展示的是一块独立的拼图块，用户需要将其滑动到与背景图片匹配的位置。组件提供了进度条显示滑动的进度，还提供了目标位置指示器帮助用户找到正确答案。

组件支持平滑的动画效果，滑动时会实时更新拼图块的位置。组件会自动记录滑动轨迹，用于后端验证和风控分析。组件提供了刷新按钮，用户可以获取新的验证码进行尝试。

### 旋转验证码组件

RotateCaptcha 组件实现了图像旋转验证功能。用户需要将图像旋转到指定的角度来完成验证。组件展示了一个圆形容器中的图像，用户可以通过拖动或点击按钮来旋转图像。组件实时显示当前旋转角度，帮助用户调整到正确位置。

组件支持精确的角度检测，可以设置目标角度进行匹配。组件提供了左右旋转按钮，方便用户进行微调。组件会自动记录旋转过程中的角度变化，用于后端验证。

### 文字验证码组件

TextCaptcha 组件实现了传统的文字输入验证功能。组件显示一组扭曲的文字或数字，用户需要正确输入这些字符。组件支持多种字符类型，包括纯数字、纯字母和混合类型。组件会自动添加干扰效果，包括背景噪点和曲线，提高安全性。

组件提供了输入框和验证按钮，用户输入完成后点击验证提交。组件支持大小写敏感和不敏感两种模式。组件会自动刷新验证码，用户可以点击刷新按钮获取新的验证码。

### 图标验证码组件

IconCaptcha 组件实现了图标选择验证功能。组件显示多个图标，并根据提示要求用户选择符合条件的所有图标。组件支持网格布局，可以自定义显示的列数和行数。组件会在用户选择时显示高亮效果，帮助用户识别已选项。

组件支持自定义选择数量限制，默认需要选择 3 个图标。组件会在验证成功后显示成功提示，验证失败后显示错误信息并自动重置。组件提供了刷新功能，用户可以获取新的图标集进行尝试。

## 示例应用

项目根目录下有一个完整的示例应用，位于 example 目录中。该示例应用展示了 SDK 的各种功能和使用方法。

运行示例应用需要先安装项目依赖，然后运行启动命令。示例应用的主界面分为左右两部分，左侧是验证码类型选择列表，右侧是验证码展示区域。应用顶部显示了当前验证码类型的名称和描述。应用底部提供了验证、重置和刷新按钮，以及验证结果显示区域。

示例应用支持以下操作流程：首先，从左侧列表选择一个验证码类型；然后，应用会加载对应的验证码组件；接着，用户进行验证操作；最后，点击验证按钮提交验证请求。应用会在底部显示验证结果，包括成功或失败状态和相关数据。

示例应用的代码结构清晰，注释详尽，非常适合作为开发参考。开发者可以通过阅读示例代码快速掌握 SDK 的使用方法，并根据需要进行定制和扩展。

## 桌面集成功能

### 系统托盘

CaptchaX Electron SDK 支持系统托盘功能，应用可以在后台持续运行而不占用任务栏空间。当用户关闭主窗口时，应用会自动最小化到系统托盘而不是完全退出。系统托盘会显示应用图标，并提供右键菜单提供快捷操作。

右键菜单包含显示窗口、新建验证码和退出三个选项。显示窗口选项可以重新显示主窗口，新建验证码选项可以直接打开验证码界面，退出选项可以完全关闭应用。双击托盘图标也可以显示主窗口。

### 原生菜单

SDK 实现了完整的原生菜单系统，菜单包含文件、编辑、视图、窗口和帮助五个主菜单项。文件菜单包含新建验证码窗口和退出选项。编辑菜单包含标准的复制、粘贴等编辑功能。视图菜单包含重新加载、开发工具等视图控制功能。窗口菜单包含最小化、最大化等窗口控制功能。帮助菜单包含关于应用和打开开发工具选项。

菜单系统与应用程序逻辑紧密集成，提供了便捷的操作入口。菜单项都配备了快捷键，方便键盘操作。菜单会根据当前状态动态启用或禁用相关选项。

### 系统通知

CaptchaX Electron SDK 支持系统通知功能，可以在验证完成时向用户发送通知。通知功能在验证成功或失败时特别有用，可以及时告知用户验证结果而无需切换窗口。

通知系统会自动检测平台支持情况，在不支持通知的平台上会优雅降级。通知支持点击事件，点击通知可以激活主窗口。通知的标题和内容都可以自定义，满足不同的业务需求。

### 开发者工具

SDK 集成了 Electron 的开发者工具支持。在开发模式下，可以直接通过菜单或快捷键打开开发者工具。开发者工具对于调试应用程序非常有帮助，可以检查 DOM 结构、查看控制台日志、分析网络请求等。

开发者工具还可以用于查看 SDK 的运行状态和日志输出。SDK 使用 electron-log 模块记录详细的运行日志，包括请求信息、响应数据、错误信息等。这些日志对于排查问题和优化性能非常有价值。

## 配置管理

### API 配置

SDK 的 API 配置是使用 SDK 的基础。默认配置指向本地开发服务器地址 http://localhost:3000。在生产环境中使用时，需要将 API 地址修改为实际的服务器地址。可以通过 SDK 实例的 setApiBaseUrl 方法动态修改 API 地址。

API 版本管理也是配置的一部分，默认使用 v1 版本的 API。SDK 会自动将版本号拼接到 API 路径中，如 /api/v1/captcha/slider。如果需要使用其他版本的 API，可以在配置中指定。

### 超时和重试

SDK 内置了请求超时和自动重试机制。默认请求超时时间为 10 秒，在网络较慢的环境中可以适当增加超时时间。超时后会抛出错误，开发者可以捕获错误并进行相应处理。

SDK 的自动重试机制会在请求失败时自动重试，最多重试 3 次。每次重试的间隔时间会递增，以避免对服务器造成过大压力。重试机制可以通过配置进行自定义，包括最大重试次数和重试间隔时间。

### 平台适配

SDK 会自动检测当前运行平台并应用相应的适配。平台信息可以通过 getPlatform 方法获取，返回值为 win、mac 或 linux。不同的平台可能会有细微的 UI 差异，SDK 会自动处理这些差异以保证用户体验。

平台特定的功能也会根据平台可用性自动启用或禁用。例如，系统托盘功能在某些 Linux 发行版上可能不可用，SDK 会自动检测并相应调整功能。

## 错误处理

SDK 的错误处理机制设计完善，能够捕获和处理各种异常情况。网络请求失败是最常见的错误类型，SDK 会自动进行重试，如果重试仍然失败则抛出错误。验证码过期是另一个常见问题，SDK 提供了 isExpired 方法来检测验证码状态。

所有错误都会通过事件系统通知开发者。ERROR 事件会携带详细的错误信息，包括错误类型、错误消息和堆栈跟踪。开发者可以根据错误信息进行相应的处理，如显示错误提示或记录日志。

在验证失败时，SDK 会通过 FAIL 事件通知开发者，并携带失败的原因。这些信息对于用户体验和后续处理都非常有帮助。开发者可以根据失败原因提供更有针对性的提示和引导。

## 最佳实践

### 用户体验优化

为了提供最佳的用户体验，建议遵循以下实践。首先，验证码加载时应该显示友好的加载提示，避免用户等待时感到困惑。其次，验证失败时应该提供清晰的错误提示，并引导用户如何正确操作。第三，验证成功后应该及时反馈，并自动进入下一步流程。

对于需要用户操作的验证码，应该提供清晰的指引和提示。提示信息应该简洁明了，避免过于技术化。同时，应该提供刷新功能，让用户可以重新尝试。

### 安全性考虑

虽然验证码本身是安全机制，但在使用时仍需注意一些安全事项。首先，不应该在客户端存储敏感的验证数据，所有验证逻辑都应该在服务端完成。其次，应该使用 HTTPS 连接进行 API 通信，避免数据被窃取。第三，应该对用户输入进行验证，避免注入攻击。

SDK 的轨迹数据包含了用户的操作行为，这些数据对于风控分析非常有价值。建议将轨迹数据完整地发送到服务端进行进一步分析。同时，应该记录验证的次数和时间，对于异常的验证行为进行监控。

### 性能优化

为了保证应用性能，建议遵循以下优化实践。首先，避免同时加载多个验证码，只加载当前需要的验证码。其次，在不需要验证码时应该及时销毁组件，释放内存。第三，使用 CDN 或本地缓存来加载验证码图片，减少网络请求。

SDK 的组件都采用了高效的渲染策略，不会造成性能问题。但在大规模使用时，仍需注意资源的合理分配。建议在应用启动时初始化 SDK，而不是在需要时才创建实例。

## 故障排除

### 常见问题

在使用 SDK 的过程中，可能会遇到一些常见问题。验证码无法加载通常是由于 API 地址配置错误或网络连接问题导致的。检查控制台日志可以获取详细的错误信息。验证总是失败可能是由于服务端验证逻辑问题，应该检查服务端日志。组件显示异常可能是由于 CSS 样式冲突，应该检查页面样式表。

通知功能不工作可能是由于系统不支持通知或通知权限被禁用。系统托盘不显示可能是由于平台不支持或图标资源缺失。开发者工具无法打开可能是由于打包后的应用禁用了开发工具。

### 日志分析

SDK 使用 electron-log 模块记录详细的运行日志。日志文件通常位于用户数据目录下的 logs 文件夹中。日志包含多个级别，包括 info、warn、error 等。通过分析日志可以定位大部分问题。

在调试时，建议开启详细日志级别以获取更多信息。生产环境中可以适当降低日志级别以减少磁盘空间占用。日志会记录所有的 API 请求和响应，方便追踪问题。

### 调试技巧

调试 SDK 时可以使用以下技巧。首先，使用开发者工具的 Console 面板查看 SDK 的日志输出。其次，使用 Network 面板查看 API 请求和响应。第三，使用 Elements 面板检查验证码组件的 DOM 结构。

在 Electron 中，可以通过设置 ELECTRON_ENABLE_LOGGING 环境变量来启用详细日志。同时，可以通过设置 ELECTRON_DISABLE_SANDBOX 环境变量来禁用沙箱模式，便于调试。
