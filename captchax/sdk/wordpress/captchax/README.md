# CaptchaX WordPress 插件

## 插件简介

CaptchaX 是一款现代化的 WordPress 行为验证码插件，为网站提供全面的安全保护。本插件支持多种验证方式，包括滑块验证、点选验证、拼图验证、旋转验证和文字验证等，有效防止恶意注册、垃圾评论和暴力破解等安全问题。

CaptchaX 采用先进的风险控制算法，能够智能识别真实用户与机器人的行为差异，在保障安全性的同时提供流畅的用户体验。插件界面简洁美观，支持自定义主题，可完美融入各类 WordPress 网站。

### 核心特性

本插件具有以下核心功能：多场景表单保护支持、完整的 WooCommerce 集成、多语言国际化支持、REST API 接口、AJAX 无刷新验证、简码嵌入功能以及深度 Contact Form 7 兼容。

### 技术架构

CaptchaX 基于 WordPress 插件标准规范开发，遵循 WordPress 5.0+ 兼容性要求，支持 PHP 7.4 及以上版本。插件采用面向对象编程设计，通过类加载器模式实现模块化管理，便于维护和扩展。所有验证码验证均通过安全的签名机制与后端 API 通信，确保数据传输的安全性。

## 功能特性

### 表单保护支持

CaptchaX 支持在多种 WordPress 表单场景中嵌入验证码验证。登录表单保护可在用户登录时进行人机验证，有效防止暴力破解攻击；注册表单保护能够阻止恶意批量注册行为；评论表单保护则可杜绝垃圾评论的自动发布；密码找回表单和密码重置表单同样受到保护，确保账户安全。

对于自定义表单场景，插件提供了简码 `[captchax]` 和 Contact Form 7 集成支持，开发者可以灵活地在任何表单中集成验证码功能。

### 验证码类型

插件支持五种主流验证码类型。滑块验证通过拖动滑块完成拼图来验证用户，操作简单直观；点选验证要求用户按顺序点击指定图片区域，适合移动端使用；拼图验证需要将缺失的拼图块拖入正确位置，安全性较高；旋转验证要求用户将图片旋转至正确角度；文字验证则让用户输入显示的字符或计算简单数学题。

### 主题定制

CaptchaX 提供浅色和深色两种主题模式，管理员可根据网站整体设计风格选择合适的主题。验证码组件的样式经过精心设计，能够自适应不同屏幕尺寸，在移动设备上同样表现出色。

### 多语言支持

插件内置完整的中英文语言包，默认支持简体中文界面。英文用户可切换至英文界面使用，所有提示文字均可通过国际化函数进行翻译，满足全球化网站需求。

## 系统要求

### 环境要求

CaptchaX 对服务器环境有明确要求以确保正常运行。PHP 版本需满足 7.4 或更高版本，以支持现代化的语法特性和安全改进；WordPress 版本需为 5.0 或更高，充分利用 WordPress 的现代功能；Web 服务器推荐使用 Apache 2.4+ 或 Nginx 1.18+，并正确配置 URL 重写规则。

### 可选依赖

WooCommerce 集成功能需要 WooCommerce 插件版本 3.0 或更高。Contact Form 7 集成功能需要 Contact Form 7 版本 5.0 或更高。这些依赖均为可选，不影响核心验证码功能的使用。

### 网络要求

服务器需能够访问 CaptchaX 验证服务器 API 地址。在开发环境中使用 http://localhost:3000，在生产环境中使用 https://captchax.example.com。确保防火墙未阻止出站 HTTPS 请求。

## 安装指南

### 自动安装

通过 WordPress 后台插件安装界面安装是推荐的方式。登录 WordPress 管理后台，进入“插件”菜单，点击“安装插件”，在搜索框中输入“CaptchaX”，找到插件后点击“立即安装”。安装完成后点击“启用插件”即可。启用后需在设置页面配置 API 密钥后方可正常使用。

### 手动安装

对于无法使用自动安装的场景，可采用手动方式安装插件。首先下载插件压缩包并解压，获得 captchax 文件夹。然后通过 FTP 或主机控制面板将该文件夹上传至 WordPress 的 wp-content/plugins/ 目录。最后登录 WordPress 后台，在插件列表中找到 CaptchaX 并点击“启用”。

### 高级配置安装

在某些特殊部署场景下，可能需要调整插件的默认配置。可以通过在 wp-config.php 中定义常量来覆盖默认设置，例如定义 `CAPTCHAX_API_KEY` 和 `CAPTCHAX_API_SECRET` 来预设 API 凭证，或定义 `CAPTCHAX_SERVER_URL` 来指定自定义的验证服务器地址。

## 配置说明

### 获取 API 凭证

使用 CaptchaX 插件前，需要在 CaptchaX 官网注册账号并创建应用以获取 API Key 和 API Secret。登录 CaptchaX 控制台后，在应用管理页面创建新应用，系统会为每个应用分配唯一的 App ID（即 API Key）。点击查看详情获取 API Secret，请妥善保管这些凭证信息。

### 后台设置页面

插件启用后，在 WordPress 后台左侧菜单中找到“设置”下的“CaptchaX”子菜单进入设置页面。设置页面分为多个配置区域，分别是常规设置区、表单配置区、WooCommerce 配置区和联系表单配置区。

### API 配置

在 API 配置区域，需要填写 API Key 和 API Secret，这两个字段为必填项。API Key 对应后台的 App ID，API Secret 为安全密钥。服务器地址字段用于指定验证服务器的 URL，开发环境使用 http://localhost:3000，生产环境使用 https://captchax.example.com。

### 验证码配置

验证码类型下拉菜单用于选择默认的验证码形式，可选值包括 slider（滑块）、click（点选）、puzzle（拼图）、rotate（旋转）和 text（文字）。主题选择确定验证码界面的配色方案，可选浅色或深色主题。语言设置控制验证码界面的显示语言。

### 表单启用配置

表单启用区域列出所有支持的表单类型，通过勾选启用相应的表单保护。可启用的表单包括评论表单、登录表单、注册表单、找回密码表单、重置密码表单和联系表单。启用后，系统会在对应表单提交时自动进行验证码验证。

### 错误消息配置

可以自定义验证码错误提示消息。错误消息字段设置用户未完成验证时的提示文本，验证失败消息字段设置验证服务器返回失败时的提示文本。这些消息支持国际化，可在翻译文件中自定义。

### WooCommerce 配置

当检测到 WooCommerce 插件已安装时，设置页面会显示 WooCommerce 专用配置区域。可分别控制结账页面、登录页面和注册页面的验证码启用状态，建议至少在结账和注册页面启用验证码保护。

### 联系表单配置

Contact Form 7 联系表单集成需要额外配置。在设置中启用联系表单验证码后，还需在 CF7 表单编辑器中添加验证码显示代码。验证码会自动附加到所有联系表单末尾，或可通过添加隐藏字段 `captchax_token` 进行手动控制。

## 使用方法

### 基础使用

完成配置后，CaptchaX 会自动在已启用的表单中显示验证码组件。用户访问登录、注册等页面时，验证码会自动加载。提交表单时，系统会先验证验证码是否通过，未通过验证的请求将被拦截并提示用户完成验证。

### 简码使用

在文章、页面或小工具中可以使用简码嵌入验证码。基础简码 `[captchax]` 会在当前位置显示验证码。更详细的配置可以通过属性参数指定，例如 `[captchax scene="custom" theme="dark" type="click"]`，可指定场景名称、主题和验证码类型。

### 开发者接口

对于需要深度定制的开发者，CaptchaX 提供了多个编程接口。通过实例化 CaptchaX_API 类并调用 verify 方法，可以在代码中执行自定义的验证码验证。通过 REST API 端点 `/wp-json/captchax/v1/verify` 可以从外部系统发起验证请求。

### JavaScript API

前端 JavaScript 提供了 CaptchaXFront 全局对象用于控制验证码行为。通过调用 `CaptchaXFront.initializeAllCaptchas()` 可以初始化页面上所有验证码组件。`CaptchaXFront.verifyManually(scene, callback)` 方法允许手动触发特定场景的验证。`CaptchaXFront.reset(scene)` 和 `CaptchaXFront.resetAll()` 方法可重置验证码状态。

### 事件钩子

CaptchaX 在验证过程中会触发多个 jQuery 事件。`captchaxVerified` 事件在验证成功时触发，回调参数包含场景名和令牌。`captchaxError` 事件在验证出错时触发。`captchaxClose` 事件在用户关闭验证码界面时触发。可通过监听这些事件实现自定义业务逻辑。

## WooCommerce 集成

### 登录保护

CaptchaX 可在 WooCommerce 的账户登录页面添加验证码保护。启用此功能后，客户在登录时会先完成人机验证。验证码显示在用户名密码输入框之后，提交按钮之前。通过设置中的 WooCommerce 登录开关可控制此功能的启用状态。

### 注册保护

新客户注册页面同样可以受到 CaptchaX 保护。启用后，注册表单会显示验证码组件，系统会在处理注册请求前验证用户身份。这对于防止虚假账户注册和批量注册攻击非常有效。

### 结账保护

WooCommerce 结账流程是电商网站最需要保护的环节。CaptchaX 在结账页面的客户信息区域前添加验证码验证。结账保护支持两种模式：完整模式和简化模式。完整模式在页面顶部显示完整验证码界面，简化模式在订单提交按钮前显示简短提示。

### 密码找回

CaptchaX 还支持 WooCommerce 的密码找回流程。当客户请求重置密码时，验证码会出现在表单中，确保密码重置请求来自真实用户而非自动化攻击。

## REST API

### 接口概览

CaptchaX 提供完整的 REST API 接口，支持第三方系统集成。API 采用 WordPress REST API 标准实现，符合 RESTful 设计规范。所有接口均支持 JSON 格式的数据交换。

### 验证接口

POST 请求到 `/wp-json/captchax/v1/verify` 可进行令牌验证。请求体需包含 token 字段（验证码返回的令牌）和 scene 字段（场景标识）。接口返回验证结果，包括 success 状态码和可选的 error 信息。

### 设置接口

GET 请求到 `/wp-json/captchax/v1/settings` 返回当前 CaptchaX 配置信息，包括是否启用、App ID、脚本地址、主题、语言和已启用的表单列表等。这对于前端动态配置验证码组件非常有用。

### 状态接口

GET 请求到 `/wp-json/captchax/v1/status` 返回插件运行状态，包括是否已配置、WooCommerce 是否可用、插件版本和服务器地址等。

### 脚本接口

GET 请求到 `/wp-json/captchax/v1/script` 返回 CaptchaX SDK 脚本的 URL 和对应的 App ID，可用于动态加载验证码组件。

## 安全特性

### 签名验证

所有与验证服务器的通信都使用 HMAC-SHA256 签名机制。请求包含时间戳和签名字段，服务器会验证签名有效性并检查时间戳新鲜度，防止重放攻击。API Secret 仅存储在服务器端，不会暴露给客户端。

### 频率限制

CaptchaX 在服务器端实现了请求频率限制机制。单个 IP 在短时间内的验证请求次数受到限制，超出限制的请求会被临时拒绝。这有效防止了暴力破解和拒绝服务攻击。

### 错误处理

插件实现了完善的错误处理机制，包括参数验证、异常捕获和日志记录。当验证失败时，用户会看到友好的错误提示而非技术细节。敏感的错误信息仅记录到服务器日志中。

## 故障排除

### 验证码不显示

如果验证码组件在页面上没有显示，首先检查插件是否已启用且配置正确。确认 API Key 和 API Secret 已正确填写，服务器地址可正常访问。检查浏览器控制台是否有 JavaScript 错误，确保网站主题没有禁用 jQuery。

### 验证始终失败

验证始终失败通常是因为 API 凭证配置错误。请仔细核对 API Key 和 API Secret 是否与 CaptchaX 控制台中的信息一致。检查服务器时间是否准确，时间偏差过大会导致签名验证失败。查看 WordPress 调试日志获取详细错误信息。

### 与其他插件冲突

某些插件可能与 CaptchaX 产生 JavaScript 冲突。尝试暂时禁用其他表单、安全或验证码相关插件进行排查。确保网站使用的 jQuery 版本为 1.7 或更高。

### 自定义表单不生效

如果自定义表单中验证码不工作，请确保表单提交方式为标准 HTTP POST，且包含名为 captchax_token 的隐藏字段。可在表单提交事件中调用 `CaptchaXFront.verifyManually()` 进行手动验证。

## 开发者指南

### 钩子与过滤器

CaptchaX 提供了多个 WordPress 钩子供开发者扩展。`captchax_enabled_forms` 过滤器可动态修改启用的表单列表。`captchax_verify_result` 过滤器可修改验证结果。`captchax_before_render` 和 `captchax_after_render` 动作钩子可在验证码渲染前后执行自定义代码。

### 数据库选项

插件使用 WordPress Options API 存储配置，主要选项键包括 captchax_api_key（API 密钥）、captchax_api_secret（API 密钥）、captchax_server_url（服务器地址）、captchax_enabled_forms（启用的表单数组）和 captchax_theme（主题设置）。

### 缓存考虑

如果网站使用了页面缓存插件，请确保验证码相关页面排除缓存或使用片段缓存。登录、注册等页面不应启用完整页面缓存，否则验证码可能无法正常工作。
