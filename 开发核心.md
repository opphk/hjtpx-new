# HJTPX 开发核心文档

## 项目概述

本项目是一个现代化的全栈应用程序，提供 RESTful API 接口服务。

## 技术栈

- 后端框架：Express.js ^4.18.2
- 数据库：PostgreSQL ^8.11.0
- 缓存：Redis ^4.6.0
- 认证：JWT ^9.0.0
- 验证：Joi ^17.11.0
- 限流：express-rate-limit ^7.1.5
- 密码加密：bcrypt ^5.1.1
- 测试：Jest ^29.7.0
- 日志：Winston
- 文件上传：Multer
- 代码格式化：Prettier ^3.0.0
- 代码检查：ESLint ^8.50.0
- Git Hooks：Husky
- CI/CD：GitHub Actions

## API 接口开发详情

### 已完成的 API 功能

#### 1. API 版本控制

- 目录：`src/backend/routes/v1/`
- 文件：`src/backend/routes/v1/index.js`
- 功能：
  - 路由版本控制
  - API 路由聚合
  - 端点注册

#### 2. 中间件体系

| 中间件 | 文件路径 | 功能 |
|--------|----------|------|
| 响应格式化 | `src/backend/middleware/responseFormatter.js` | 统一响应格式 |
| 日志记录 | `src/backend/middleware/logger.js` | 请求/响应日志 |
| 请求日志 | `src/backend/middleware/requestLogger.js` | Winston 请求日志 |
| 限流控制 | `src/backend/middleware/rateLimiter.js` | IP/用户级别限流 |
| CORS | `src/backend/middleware/cors.js` | 跨域资源共享 |
| 数据验证 | `src/backend/middleware/validator.js` | Joi 验证 |
| 错误处理 | `src/backend/middleware/errorHandler.js` | 统一错误处理 |
| 文件上传 | `src/backend/middleware/upload.js` | Multer 文件上传 |

#### 3. 验证规则

- 文件：`src/backend/utils/validationRules.js`
- 规则：
  - `userSchema`：用户注册验证
  - `loginSchema`：登录验证
  - `updateUserSchema`：用户更新验证

#### 4. API 端点

**健康检查**

- `GET /api/v1/health` - 基础健康检查
- `GET /api/v1/health/detailed` - 详细健康检查

**认证接口**

- `POST /api/v1/auth/login` - 用户登录
- `POST /api/v1/auth/register` - 用户注册
- `POST /api/v1/auth/verify` - 验证 Token
- `POST /api/v1/auth/refresh` - 刷新 Token
- `POST /api/v1/auth/logout` - 用户登出

**密码重置**

- `POST /api/v1/password/forgot` - 请求密码重置
- `POST /api/v1/password/reset` - 重置密码

**用户管理**

- `GET /api/v1/users` - 获取用户列表
- `GET /api/v1/users/:id` - 获取指定用户
- `POST /api/v1/users` - 创建用户
- `PUT /api/v1/users/:id` - 更新用户
- `DELETE /api/v1/users/:id` - 删除用户

**通知系统**

- `GET /api/v1/notifications` - 获取用户通知列表
- `GET /api/v1/notifications/unread-count` - 获取未读数量
- `GET /api/v1/notifications/:id` - 获取指定通知
- `POST /api/v1/notifications` - 创建通知
- `PUT /api/v1/notifications/:id/read` - 标记为已读
- `PUT /api/v1/notifications/read-all` - 全部标记为已读
- `DELETE /api/v1/notifications/:id` - 删除通知

**数据导出**

- `GET /api/v1/export/:format` - 导出数据（支持 CSV、JSON、Excel）
- `POST /api/v1/export/batch` - 批量导出

**数据导入**

- `POST /api/v1/import/csv` - 导入 CSV 文件
- `POST /api/v1/import/json` - 导入 JSON 文件
- `POST /api/v1/import/preview` - 预览导入数据

**文件管理**

- `POST /api/v1/files/upload` - 上传文件
- `GET /api/v1/files` - 获取文件列表
- `GET /api/v1/files/stats` - 获取存储统计
- `GET /api/v1/files/:id` - 获取文件详情
- `GET /api/v1/files/:id/download` - 下载文件
- `POST /api/v1/files/:id/copy` - 复制文件
- `POST /api/v1/files/:id/move` - 移动文件
- `DELETE /api/v1/files/:id` - 删除文件
- `DELETE /api/v1/files/folder/:folder` - 删除文件夹

**搜索功能**

- `GET /api/v1/search/:model` - 搜索模型数据
- `POST /api/v1/search/:model/advanced` - 高级搜索
- `GET /api/v1/search/:model/suggestions` - 获取搜索建议
- `GET /api/v1/search/:model/fields` - 获取可搜索字段

#### 5. API 文档

- 文件：`src/backend/routes/v1/docs.yaml`
